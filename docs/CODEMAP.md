# Code map — where does X live?

> **Audience:** You're about to edit something and need to find the
> file. This is the shortcut. For the *why* of a choice read
> `DECISIONS.md`; for the *how* of a convention read `CONVENTIONS.md`.
>
> Paths are relative to repo root. Click-to-open in most editors.

---

## Top-level shape

```
yarit-shop/
  src/                      ← all application code
    app/                    ← Next.js App Router (route groups)
    collections/            ← Payload CMS collection schemas
    components/             ← React components (storefront + admin)
    globals/                ← Payload globals (SiteSettings)
    lib/                    ← Helpers, utilities, integrations
    messages/               ← i18n message dictionaries (he.json, en.json)
    brand.config.ts         ← ★ single source of truth for brand data
    payload.config.ts       ← Payload CMS config (admin, DB, plugins)
    middleware.ts           ← next-intl locale + admin middleware
  docs/                     ← every doc (read INDEX.md to navigate)
  New/handoff/              ← Living Garden design prototypes
  public/                   ← static assets (brand images, fonts if any)
  .claude/                  ← Claude Code settings + launch.json
```

## Admin (Payload CMS)

### Routes — `src/app/(payload)/`

| Path | Purpose |
|---|---|
| `layout.tsx` | Admin root — loads Bellefair via next/font (independent from storefront). |
| `admin/` | Payload admin panel pages (Payload owns most of this). |
| `admin/importMap.js` | ★ Auto-generated by `npm run build`. Contains the Payload component imports. **Always check the diff after every build** — the `VercelBlobClientUploadHandler` line has been wiped twice before (P0 regressions). Restore with `git checkout HEAD -- src/app/(payload)/admin/importMap.js` if missing. |
| `admin-brand.css` | Admin-only styles (Night Apothecary palette kept here). |
| `api/` | Payload REST + GraphQL. |

### Collections — `src/collections/`

| File | Purpose |
|---|---|
| `Products.ts` | Product schema. `type: 'stocked' \| 'sourced'` discriminator controls stock-tracking visibility. |
| `Categories.ts` | Category taxonomy. |
| `Orders.ts` | Customer orders + state machine (`packed → shipped → delivered`). |
| `Media.ts` | Uploads (goes to Vercel Blob in prod, local storage in dev). |
| `Tags.ts` | Freeform tags for products. |

### Admin custom components — `src/components/admin/`

Payload admin's top-bar, sidebar, dashboard tiles, and fulfillment view
live here. The import paths from Payload config register these.

## Data + integrations

### `src/lib/`

| Path | Purpose |
|---|---|
| `payload.ts` | Shared Payload client + helper to fetch with the right permissions context. |
| `i18n/routing.ts` | next-intl routing config: locales `['he','en']`, default `he`, prefix `as-needed`. |
| `i18n/navigation.ts` | ★ Re-exports `Link / redirect / usePathname / useRouter / getPathname` — **never import these from `next/link`**. |
| `siteSettings.ts` | Resolves `getResolvedSiteSettings` with `isPlaceholder` guards on contact fields. |
| `payments/` | Payment-provider abstraction. Currently: Meshulam (pending credentials). Never import `meshulam.ts` from a route — go through `provider.ts`. |
| `email/` | Email-provider abstraction (Resend). Same pattern — import through the abstraction, not the driver. |
| `auth/` | JWT session helpers + `getCurrentUser`. |
| `cart/store.ts` | Zustand cart store + `persist` middleware. `selectCartCount` + `selectCartSubtotal` selectors. |
| `motion/gsap.ts` | ★ Single GSAP entry point. `ScrollTrigger` + `Flip` pre-registered. **Every GSAP import comes through here**, never from `gsap/ScrollTrigger` directly. |
| `motion/useGsapReducedMotion.ts` | Reactive `prefers-reduced-motion` hook (useSyncExternalStore). |
| `shipping.ts` | Israel shipping rates. |
| `checkout.ts` | Checkout orchestration + validation. |
| `orders/` | Order helpers (totals, status transitions). |
| `orderToken.ts` | Token-signed order access for logged-out confirmation viewing. |
| `legal.ts` | Legal-markdown loader. |
| `seed.ts` | Dev seed script for products + categories. |
| `format.ts` | Currency, date, locale formatters. |
| `product-image.ts` | Product image URL + fallback helpers. |
| `cn.ts` | ★ `cn()` — tiny class-name joiner. Use it everywhere conditional classes matter. |
| `useHasMounted.ts` | Client-mount detection hook for hydration-safe client-only UI. |
| `admin/` | Admin-side helpers. |

### `src/globals/SiteSettings.ts`

Payload global for site-wide settings — social links, contact info,
announcement bar copy. Edit through `/admin` not code.

## Styling + theming

| File | Purpose |
|---|---|
| `src/app/globals.css` | ★ Single storefront stylesheet. Tailwind v4 `@theme` block, Night Apothecary `--color-*` palette, Living Garden `--g-*` palette + motion + chrome rules, body ambient layers, `prefers-reduced-motion` block. **No other CSS file is loaded by the storefront.** |
| `src/app/(payload)/admin-brand.css` | Admin-only styles. Night Apothecary palette. Loaded by Payload layout, NOT the storefront. |
| `src/brand.config.ts` | ★ Brand name / tagline / description / fonts / colors. Change colors here AND in globals.css `@theme`. |
| `tailwind.config.ts` | **Does NOT define tokens.** Tailwind v4 reads `@theme` from globals.css. This file only holds plugin config + content glob. |

### Fonts

Loaded via `next/font/google` in `src/app/(storefront)/[locale]/layout.tsx`:

| Font | CSS variable | Role |
|---|---|---|
| Heebo | `--font-heebo` | Sans body (supports Hebrew + Latin) |
| Fraunces | `--font-fraunces` → `--font-display` / `--g-display` | Display serif (headings, wordmark) |
| Source Serif 4 | `--font-source-serif` → `--g-body` | Body serif |
| Caveat | `--font-caveat` → `--g-accent` | Handwritten accent (degree mark, notes) |
| JetBrains Mono | `--font-jetbrains` → `--g-mono` | Mono kicker + badges |

Admin loads Bellefair independently as `--font-frank-ruhl` via `src/app/(payload)/layout.tsx`.

## i18n

| File | Purpose |
|---|---|
| `src/messages/en.json`, `src/messages/he.json` | ★ Every UI string. Same key tree in both files. |
| `src/lib/i18n/routing.ts` | locales config |
| `src/lib/i18n/navigation.ts` | Link + router exports (use these, not `next/link`) |
| `src/middleware.ts` | next-intl middleware — handles locale detection + admin path guards |

**Rule:** add a string → add to both `he.json` AND `en.json` → import via `useTranslations(namespace)` or `getTranslations(namespace)`.

## Motion system

| File | Purpose |
|---|---|
| `src/lib/motion/gsap.ts` | Single GSAP entry (+ ScrollTrigger + Flip pre-registered) |
| `src/lib/motion/useGsapReducedMotion.ts` | `prefers-reduced-motion` hook |
| `src/components/motion/GsapScope.tsx` | `useGsapScope` wrapper — use this for every GSAP client component |
| `src/components/motion/GardenAlive.tsx` | Living Garden cursor + leaf trail + scroll vine + card parallax |
| `src/components/motion/RevealOnScroll.tsx` | GSAP ScrollTrigger adapter for `.g-reveal` elements |
| `src/components/motion/Reveal.tsx`, `StaggeredReveal.tsx` | IntersectionObserver-based reveals (predecessor to RevealOnScroll — still in use on non-Living-Garden pages) |
| `src/components/motion/KenBurns.tsx`, `SplitWords.tsx`, `CountUp.tsx`, `ConfettiTrigger.tsx` | Specific motion patterns |
| `src/components/ui/DriftingLeaves.tsx` | 5-SVG-leaf drifting background layer, scroll-responsive via GSAP velocity |

**Invariant (CLAUDE.md rule #12):** every `gsap.from + scrollTrigger` uses `immediateRender: false + once: true + start: 'top bottom-=40'`.

## Rules of thumb when editing

1. **Find the file via this map → read it whole → edit.** Don't skim.
2. **Strings → messages files.** No hardcoded UI copy.
3. **Theme change → `brand.config.ts` + `globals.css` both.**
4. **Motion → use `useGsapScope` wrapper.**
5. **Payment/email → go through the provider abstraction.**
6. **Admin change → re-check `importMap.js` after every build.**
7. **Storefront link → `Link` from `@/lib/i18n/navigation`.**
